# Qlib安全統合計画

更新日: 2026-07-22

## 結論

Microsoftのオープンソースプロジェクト `microsoft/qlib` を、予測値を直接信用する売買システムではなく、特徴量生成・機械学習・時系列検証・モデル比較の研究基盤として限定利用する。

- 公式リポジトリ: https://github.com/microsoft/qlib
- ライセンス: MIT
- データ: 現在のJ-Quantsおよび保存済み株価だけを利用
- Qlibのコミュニティ配布データや未知の学習済みモデルは取り込まない
- 自動売買は実装せず、既存LINE通知の参考スコアとして比較する

## セキュリティ条件

1. 専用の隔離仮想環境へ導入する。
2. PyPIの安定版をバージョン固定し、GitHubのmainブランチを直接実行しない。
3. 依存関係を `pip-audit` で検査し、結果を保存する。
4. 外部コードへAPIキー、LINEトークン、GitHubトークンを渡さない。
5. 外部データセット、pickle、未確認の学習済みモデルを読み込まない。
6. GitHub Actionsは固定バージョンのActionsだけを使用し、書き込み権限を最小化する。
7. Qlibの出力は既存システムと分離し、異常時に従来判定へ戻せるようにする。

## 無償条件

- Qlib本体はMITライセンスで無償。
- 学習はローカルPCまたはGitHub Actionsの無償枠内で行う。
- 有料データや有料LLMは利用しない。
- データ取得は現在契約中のJ-Quantsで利用可能な範囲と、利用規約に従ったyfinanceに限定する。
- GitHub Actionsの利用量が無償枠を超えないよう、日次推論と週次学習を分離する。

## 精度検証

「精度向上」はバックテストの利益率だけで判断しない。既存ルールを基準モデルとして、同じ期間・同じ銘柄群で次を比較する。

- ウォークフォワード検証
- 学習・検証・テスト期間の完全分離
- 開示日基準の財務データ利用
- 取引コストとスリッページ控除
- 年別および相場局面別の成績
- IC、Rank IC、勝率、平均リターン、最大ドローダウン
- 複数乱数シードでの安定性
- サバイバーシップバイアス、先読み、データ漏洩の検査

Qlibモデルが未使用期間で既存ルールを安定して上回らない限り、本番LINE候補の順位には反映しない。

## 導入段階

### Phase 1: 安全な比較実験

- TOPIX Core30の保存済みデータをQlib形式へ変換
- LightGBMなど説明しやすい教師ありモデルを1種類だけ使用
- 未来5営業日の相対リターンを予測対象にする
- 通知・公開・設定変更は行わない

### Phase 2: 検証強化

- 対象をTOPIX 100へ拡張
- 既存テクニカル指標、財務指標、市場レジームを特徴量化
- ウォークフォワード評価とモデル劣化検知を追加

### Phase 3: 参考スコア統合

- Qlibスコアを既存期待値スコアとは別欄でLINEへ表示
- 少なくとも一定期間は既存順位へ影響させない
- 安定性確認後にのみ、重み付け統合を別途承認する

## 今回採用しない候補

- FinRL: MITライセンスだが、公式に教育・研究用途の位置付けがあり、強化学習は依存関係と過学習リスクが大きいため初期導入しない。
- 未知の個人リポジトリ: 保守体制、セキュリティポリシー、ライセンス、再現性を確認できないため使用しない。
- 公開済み「必勝」モデルや売買シグナル: データ漏洩や誇張されたバックテストを排除できないため使用しない。
